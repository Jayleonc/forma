Forma 转换接口 API 说明文档
1. 接口对比概览
特性	/api/v1/convert (现有接口)	/api/v1/convert-by-content (新增接口)
主要用途	处理外部文件（PDF, DOCX, PPTX, 图片）	处理已有 Markdown 内容（Git Sync, 编辑器）
输入方式	source_url (远程下载链接)	markdown_content (Markdown 文本字符串)
下载步骤	需要 从 source_url 下载原始文件	不需要，直接处理请求体内容
转换逻辑	复杂转换（解析、OCR、VLM 图像识别等）	轻量规范化（行尾符、UTF-8 校验、去空字符）
适用场景	用户上传的各类文档原始素材	数据库中已存在的 Markdown 文稿
任务模型	异步任务（返回 task_id，回调告知结果）	异步任务（返回 task_id，回调告知结果）
2. 接口详细说明
A. 远程文件转换接口
Endpoint: POST /api/v1/convert

支持范围:
PDF 文档 (包含扫描件)
Word 文档 (.docx)
PowerPoint 文档 (.pptx)
图片格式 (.png, .jpg, .jpeg, .bmp)
请求要求:
source_url: 必须是 Forma 能够访问到的 HTTP/HTTPS 直连下载链接。
callback_url: 转换完成后，结果通过 POST 推送至此地址。
处理机制:
Forma 会先下载文件到本地临时目录。
根据文件类型调用不同的解析器。
支持 auto/fast/deep 策略（deep 模式会调用 VLM 识别图片）。
B. Markdown 内容规范化接口
Endpoint: POST /api/v1/convert-by-content

支持范围:
纯文本 Markdown 内容。
请求要求:
markdown_content: 必填，Markdown 正文。
大小限制: 默认最大支持 2MB (可通过 FORMA_MAX_INLINE_MD_BYTES 配置)。超限将直接返回 400 错误。
callback_url: 必填。
处理机制:
跳过下载: 直接读取内存中的文本，不产生下载开销。
基础规范化:
移除 \x00 空字符。
统一换行符为 Unix 风格 (\n)。
去除每行末尾多余空格。
确保文末有且仅有一个换行。
安全性: 日志中仅打印内容长度和前 80 字符预览，禁止打印全文以保护数据隐私。
3. 回调说明 (Callback)
两个接口的回调格式完全兼容，旨在降低接入成本。

POST 到 callback_url 的内容:

json
{
  "request_id": "调用方传入的 ID",
  "status": "completed | failed",
  "markdown": "处理后的 Markdown 内容 (成功时)",
  "error": "错误描述信息 (失败时)"
}
注：Forma 会自动识别 Knowledge Hub 的回调链接（路径含 callback/forma/convert 或带 token 参数），并自动调整字段名以适配 KH 现有的消费逻辑。

4. 接入建议
如果你只有文件 URL: 请继续使用 /api/v1/convert。
如果你已经拿到了 Markdown 文本: 请使用 /api/v1/convert-by-content。这样可以避免 Knowledge Hub 还需要为此文本生成一个临时下载 URL，同时也能规避文件下载失败（404）导致的任务挂死问题。
